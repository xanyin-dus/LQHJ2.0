## 🎯 项目简介
LQHJ2.0（灵棋幻境）是一款跨平台五子棋游戏，核心特色是在经典玩法基础上融入**轻量剧情分支**，通过QML+C++混合开发实现UI与逻辑解耦，兼顾游戏性与叙事体验。

> ✅ **关于main分支**
> 本`main`分支是项目的**稳定集成与发布分支**，仅包含经过测试、审核的完整功能代码，可供直接体验、部署与发布。禁止所有开发者直接向本分支提交代码，功能需先在`arch`或功能分支开发完成，经审核后合并至本分支。

### ✨ 核心体验特性
- ♟️ **完整五子棋玩法**：人机AI，落子校验、胜负判定、悔棋等
- 📖 **轻量剧情系统**：多分支剧情选择，剧情与游戏玩法联动，解锁不同故事线
- 🎨 **流畅交互体验**：QML实现的动态UI与动画，支持棋盘大小、音量等个性化设置
- 💾 **本地存档功能**：自动保存游戏进度与剧情分支，支持多存档管理
- 📱 **跨平台支持**：暂时只兼容Windows，期待Linux、macOS的开发多端运行

## 🚀 快速开始
### 前置环境
1. 安装 **Qt 6.8**（需包含QML、Qt Quick、Qt Core模块）
2. 安装 **CMake 3.16+**
3. 安装对应平台编译器（Windows：MSVC2019+/MinGW；暂未支持：Linux：GCC；macOS：Clang）

### 克隆与运行
```bash
# 克隆仓库
git clone https://github.com/xanyin-dus/LQHJ2.0.git LQHJ20
cd LQHJ20

# 编译运行
# 1. 打开Qt Creator，导入项目根目录的CMakeLists.txt
# 2. 配置Qt 6.8与对应编译器，构建并运行
```

## 🔒 主分支维护规范
### 1. 分支管理规则
| 分支类型   | 权限与流程                                                                 |
|------------|--------------------------------------------------------------------------|
| `main`     | 仅项目负责人可合并，禁止直接提交代码。功能需先在`arch`/功能分支开发完成，经审核后合并 |
| `arch`     | 架构基准分支，功能分支需基于此分支开发，合并至`main`前需先合并到`arch` |
| 功能分支   | 基于`arch`创建，开发完成后向`arch`发起PR，审核通过后再合并至`main` |

### 2. 合并流程
1. 功能分支开发完成 → 自测通过 → 向`arch`发起PR
2. 架构分支审核通过 → 合并至`arch`
3. 项目负责人定期将`arch`的稳定代码合并至`main`
4. `main`分支发布版本时打标签

## ❓ 常见问题
### Q：主分支代码和arch分支有什么区别？
A：`arch`是**架构基准分支**，仅包含基础框架与通用组件；`main`是**稳定发布分支**，包含所有已完成的功能代码，可供直接体验。

### Q：如何向主分支贡献代码？
A：需先基于`arch`创建功能分支，开发完成后向`dev-1.0.0`发起PR，审核通过后由项目负责人合并至`main`。

### Q：主分支编译失败怎么办？
A：`main`分支为稳定版本，若编译失败可尝试：
1. 确认Qt版本为6.8且模块完整
2. 清空构建目录，重新配置CMake
3. 拉取最新代码：`git pull origin main`
