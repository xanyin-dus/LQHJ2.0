# 🎮 灵棋幻境- Doc分支说明

本分支仅用于**集中管理项目所有文档**，无任何代码

## 一、项目&分支简介 📌

项目：基于QML+C++开发的「灵棋幻境」，兼顾休闲玩法与轻量化剧情体验。

Doc分支：独立文档仓库，整合项目全流程文档，实现统一管理、版本可控、快速检索。

## 二、文档目录 📂

```plain text
doc/
├── README.md                  # 本分支说明
├── 01_需求文档                # 用户/产品需求、需求变更记录
├── 02_设计文档                # 架构、界面、剧情、算法设计--待上传
├── 03_开发文档                # 环境搭建、编码规范、常见问题--待上传
├── 04_测试文档                # 测试计划、用例、报告--待上传
├── 05_部署文档                # 本地运行、打包指南--待上传
└── 06_归档文档                # 历史/废弃文档归档--待上传
```

## 三、文档维护规范 ✅

**命名规范**

- 中文全称+清晰后缀

- 无特殊字符、空格，避免乱码

**更新规范**

- 新增/修改后，同步更新目录索引

- 删除文档需迁移至归档目录

**格式规范**

- 自由格式（md、pdf、word）

- 标题层级清晰，关键信息加粗

**版本规范**

- Feature分支提交，MR/PR审核后合并

- Commit信息简洁明了

## 四、快速使用指南 🚀

1. 克隆分支：`git clone -b doc [项目仓库地址]`

2. 查阅文档：进入对应目录，参考子目录README索引

3. 提交修改：按规范创建分支→提交→发起MR/PR
